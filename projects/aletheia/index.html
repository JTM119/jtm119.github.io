Section 1

Overview & Motivation

Project Aletheia is a Phase II STTR software tool for detecting manipulation in raster digital images (JPEG, PNG, etc.). Digital images are a primary intelligence source and a prime target for adversarial manipulation. Skilled editors can hide modifications from human reviewers, especially when analysts lack ground truth.

A computer-based tool offers four critical advantages over human review:

  • Pixel-level detection of compression artifacts and noise patterns imperceptible to human eyes
  • Spatial localization of manipulated regions on a pixel-by-pixel basis
  • Manipulation type classification (inpainting, copy-move, splicing)
  • High throughput — hundreds of images analyzed while a human processes one

The project was structured across seven milestones: project kickoff, dataset acquisition, CNN architecture selection, model training, validation and verification, front-end deployment, and final reporting.

Scope note: Retouching (filters, color grading, sharpening) and source masking (camera device fingerprint hiding) were deliberately excluded. These manipulations do not alter the semantic content of an image and pose minimal intelligence risk.
AI-generated content was outside the original Phase II scope, but VISIMO conducted additional R&D to address this growing threat. Results are covered in Section 5.

Section 2

Manipulation Methods

Aletheia targets three categories of manipulation that alter the semantic content of an image. Each leaves a distinct forensic signature in the pixel and compression domain.

Inpainting

Merging large numbers of neighboring pixels or texture patches to modify a region — using surrounding colors and patterns to fill or remove content. Analogous to digital cloning or brush tools.

Example: Removing the rope holding a person aloft to make them appear to levitate.

repetition artifacts unnatural textures

Copy-Move

Copying one or more semantic regions within the same image and pasting them elsewhere, typically to duplicate content (e.g., enlarging a flock of birds).

Example: Duplicating aircraft in a formation to exaggerate fleet size.

edge blending artifacts noise inconsistency

Splicing

Copying a region from a separate donor image and inserting it into the target. The donor and host may have different camera sensors, lighting, and compression histories.

Example: Pasting a weapon system into a benign scene from a different photograph.

DCT coefficient mismatch noise level mismatch

CAT-Net Architecture

CAT-Net (Compression Artifact Tracing Network) is a deep CNN with two parallel processing branches that are fused to produce a binary manipulation mask.

  • RGB branch: Analyzes raw pixel values for visual inconsistencies
  • DCT/Q-Table branch: Analyzes JPEG compression coefficients and quantization tables for statistical anomalies introduced at re-compression
  • Stages: Computational blocks operating at fixed resolution
  • Transitions: Upsample/downsample blocks for multi-scale analysis
  • Output: Binary pixel mask (0 = authentic, 1 = manipulated)
Why three CAT-Net instances? Training neural networks has a stochastic element; each trained instance produces unique "hallucination" errors. Running three independently trained models and combining them via deep consensus suppresses these per-model artifacts without requiring architectural changes.

Deep Consensus Formula

For each pixel, outputs from all three CAT-Net models are aggregated into a single score in [0, 1]:

  • Score ≈ 0: Authentic prediction
  • Score ≈ 1: Manipulation prediction
  • Score ≈ 0.5: Ambiguous — no decisive vote

Secondary Classification Network (SCN)

The SCN takes the deep consensus mask and a modified version of the original image as inputs. It outputs four class probabilities: authentic, inpainting, copy-move, and splicing. Trained on 30,000 images using CAT-Net outputs as labels.

Section 5

AI-Generated Content Detection

While outside the original Phase II scope, VISIMO conducted additional R&D to address AI-generated imagery. Initial testing showed the base Aletheia tool detected all AI-generated test images at 100% confidence (from DALL-E, Midjourney, and VISIMO's CGAN algorithm), but struggled to determine whether the entire image or only part of it was AI-generated — motivating a dedicated model.

CLIP Model Approach

The CLIP (Contrastive Language-Image Pre-Training) model was fine-tuned to classify images as real or AI-generated. Image-caption pairs are fed through separate image and text encoders to create embeddings in a shared space; similarity between embeddings determines classification. Weights are learned via ADAM optimization minimizing combined cross-entropy loss across image and text modalities.

Interpretation: The model reliably detects whether manipulation occurred (98.3% precision, 97.4% recall). Pixel recall of 81.2%+ means the model finds the majority of manipulated pixels. Precision is lower because the model tends to extend its mask slightly beyond the true boundary. All pixel kappa scores (>0.67) indicate agreement far beyond random chance. The classifier struggles to distinguish which type of manipulation occurred (53% overall accuracy), which is a harder problem — the binary authentic/manipulated split is the more operationally relevant metric.

Model Retraining with IMD2020 Extended

The DEFACTO dataset's algorithmic manipulation generation introduces systematic biases: spliced objects are restricted to context-invariant shapes; copy-move duplicates only along cardinal axes; inpainting focuses on simple backgrounds. Retraining with the IMD2020 Extended dataset (human-made manipulations, 32+ camera types, GAN inpainting) was performed to reduce sensitivity to these artifacts.

User Feedback Summary

NASIC test users completed structured surveys after using the deployed application. Analysis was conducted on batches of 1–5 images across multiple formats (JPG, JFIF, GIF, TIF, PNG, BMP, WEBP).

Strengths

  • Application loaded quickly; upload and login workflows were clear
  • WebSocket live updates worked correctly; prior-session results persisted on re-login
  • Image mask overlays were visually clear and useful
  • Analysis of 4–5 image batches completed in 1–3 minutes
  • Detection sensitivity slider was appreciated
  • CSV export clearly identified predicted manipulation type

Areas for Improvement

  • Upload accepts only .zip files; direct image upload was requested
  • CSV probabilities in scientific notation (e.g., 4.37e-21) instead of percentages
  • CSV export opened in same tab, requiring browser back button to return
  • GIF files and some WEBP/TIF files caused failures or unexpected results
  • AI-generated / deepfake detection was the top missing feature request
  • Small images with small manipulated regions were harder to detect
Post-feedback updates: Following initial user testing, VISIMO redeployed the application with HEIC file type support, TIFF preview rendering, and a GPU resource exhaustion error message.

Section 8

Conclusions & Recommendations

All seven project objectives were successfully completed. The combined CAT-Net, Deep Consensus, and SCN pipeline demonstrates strong pixel-level manipulation detection with >97% manipulation detection recall, and the distributed web application was deployed and validated with real NASIC users.

Future Work

  • 1
    AI-Generated Detection Integration Add the fine-tuned CLIP model as a dedicated classification head within Aletheia. Requires backend work to load the CLIP model and frontend work to display real/fake classification alongside the existing manipulation mask. VISIMO has already completed initial R&D.
  • 2
    Dataset Expansion Integrate more large-scale, human-generated manipulation datasets. VISIMO is building a custom dataset with varied software tools for inpainting, copy-move, and splicing to improve robustness beyond DEFACTO's algorithmic uniformity.
  • 3
    Model Robustness for AI Content Consider supplementing CAT-Net with a model better suited to diffusion model noise patterns, which differ from the digital-photograph JPEG compression artifacts CAT-Net is optimized to detect. Diffusion-specific detectors may be more stable as new generation models emerge.
  • 4
    UX Improvements Direct image upload (no zip required), percentage-based confidence display, separate-tab CSV export, folder-level batch reporting, and expanded file type support (GIF, WEBP, TIF).
  • 5
    SSO Authentication Implement Single Sign-On via NASIC's preferred identity provider to replace local credential management before scaling beyond the current test user base. The SSO framework is already in place within the application.
  • 6
    Kubernetes Scale-Out Deployment The existing containerized architecture is already Kubernetes-compatible. Cloud or private-network scale-out deployment is achievable with minimal additional engineering effort.
