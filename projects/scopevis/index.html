The Problem

Manual parameter tuning is a scientific bottleneck

Modern nanoscale imaging at national laboratory facilities generates four-dimensional datasets: three spatial dimensions plus chemical element channels. Determining correct visualization parameters for each dataset requires enormous manual effort. ScopeVis automates this process end-to-end.

Architecture

A four-stage ML pipeline

The ScopeVis pipeline automatically processes multichannel microscopy volumes through thresholding, segmentation, ROI selection, and parameter optimization. Results surface directly in the web application as recommended starting parameters.

Fig 3 — Full ML architecture. Input volumes are segmented per-channel (purple region), ROIs and optimal channel subsets are identified (blue region), and visualization parameters are optimized via Bayesian TPE (red region).
Stage 1

Thresholding

Raw microscopy intensity values span wide ranges (0–12,000+) that are incompatible with the 0–255 RGB range required by segmentation models. Linear rescaling produces images too dark to analyze. ScopeVis uses a two-step method to find the optimal lower and upper bounds before rescaling.

Lower Bound — KDE Method

A Kernel Density Estimate is fit to the intensity histogram. The first local maximum of the KDE curve identifies the background peak, becoming the lower bound. This handles noisy backgrounds that don’t start at zero.

Fig 5 — KDE (black line) fit to the intensity histogram (grey bars). The first local maximum at 579.82 (dashed blue line) is selected as the lower bound, separating background from image content.
Reference Image

This Z-axis slice at index 22, Channel 0 is the example used throughout the thresholding section. Raw intensity range: 0–12,000+.

Fig 4 — Single Z-axis slice of a raw fluorescence microscopy channel. Too dark to analyze without thresholding.
Upper Bound — Entropy Optimization

Three metrics were evaluated: KL-Divergence, Gini Index, and Entropy. Entropy produced the best visualizations. ScopeVis iterates over 100 candidate upper bounds and selects the value maximizing a joint objective of entropy (even use of 0–255 range) and data inclusion (minimizing clipped pixels).

Fig 10 — Entropy decreases as the upper bound increases; the data inclusion factor plateaus near 60%. ScopeVis selects the upper bound at 2377 (black vertical line) that jointly maximizes both metrics.
Fig 11 — Intensity mapping curve and the final thresholded image. Data within the bounds is linearly scaled to 50–150; data outside is exponentially scaled to suppress white-flare artifacts.

3D Volume Assembly via Graph Clustering

SAM operates on 2D slices. Naive IoU threshold joining fails for diagonal or stacked objects. ScopeVis uses graph spectral clustering: each 2D segment becomes a node, overlapping segments form weighted edges (centroid distance + IoU), and spectral clustering produces coherent 3D volumes.

Fig 14 — Segmentation masks become graph nodes; nodes with spatial overlap form weighted edges; spectral clustering on the adjacency matrix yields 3D segmented volumes.
Fig 13 — Three failure modes for naive IoU joining. Left: a standard vertical cylinder where any missed slice splits the object. Center: two stacked cylinders with a gap that must remain separate. Right: a diagonal cylinder whose per-slice overlap is partial. Graph spectral clustering handles all three correctly.
Sobel-Enhanced Custom Watershed

For lower-resolution data, standard watershed produces a single mask for the entire region. ScopeVis supplements watershed with 3D Sobel edge detection: gradient magnitudes are computed as ‖∇G‖ = √(Gx² + Gy² + Gz²). Watershed then computes distance-to-edge rather than distance-to-background, enabling detection of sub-structures separated by gradual intensity transitions.

Fig 15 — Raw low-resolution slice (left) and the imperfect single-region mask produced by standard watershed and region-growing (right).
Fig 16 — Original slice (left) and the Sobel3D edge map (right). The edge map reveals structural boundaries invisible to standard intensity-based segmentation.
Fig 17 — Distinct segmentation masks produced by the Sobel-enhanced custom watershed. Multiple separate regions are correctly identified where the standard method produced a single blob.
Stage 3

Region of Interest Selection

After segmentation, cross-channel segments corresponding to the same physical object are grouped into unified ROIs. A second graph clustering algorithm identifies which segments from different chemical channels co-localize spatially, then recommends the optimal channel subset for viewing each ROI.

Fig 18 — Independent segmentation masks for each chemical channel (Co, La, Mn, Ni) in a four-element dataset. Co, Mn, and Ni show strong spatial overlap; La does not co-localize with the main body.
Fig 19 — ROI grouping result. Top row: original per-channel masks. Bottom row: unified ROI assignments — regions sharing a color across channels belong to the same physical ROI. The red ROI spans Co/Mn/Ni; the yellow ROI spans Co/La/Ni.
Stage 4

Parameter Selection

Two groups of parameters are selected per ROI: color assignments (which RGB vectors map to each channel) and non-color parameters (contrast, brightness, transparency, gamma, etc.).

Color Selection

Orthogonal RGB Vectors

Each channel is treated as a vector in 3D color space. ScopeVis assigns colors so that selected channel vectors are as orthogonal as possible, maximizing visual separability between co-visualized channels.

Non-Color Parameters

Bayesian TPE Optimization

A Tree-structured Parzen Estimator maintains two KDEs — for high-reward g(x) and low-reward l(x) configurations — and selects new candidates by maximizing g(x)/l(x). More efficient than grid or random search.

TPE Reward Function
Criterion 1

Visual Distinctness

The ROI must be visually distinguishable from its surroundings. Measured as contrast between the ROI region and the surrounding rendered image.

Criterion 2

Information Preservation

The visualization must not obscure original data features. Measured using mutual information between the rendered output and each input channel. Configurations that erase detail are penalized.

Fig 20 — Example final output from the parameter selection stage. Two channels are rendered with TPE-selected colors, contrast, and transparency, revealing the structural relationship between elements within the ROI.

The TPE pipeline runs for a configurable number of iterations. During each iteration, a candidate parameter set is applied to the image; visual distinction and mutual information are computed and fed back to TPE to update its probability models.

This optimization cannot be performed via gradient descent — the relationship between visualization parameters and perceptual quality has no differentiable form — making the Bayesian approach particularly well suited.

Key Findings
  • The entropy-based thresholding metric outperformed KL-Divergence and Gini Index for producing interpretable rescaled microscopy images.
  • Graph spectral clustering for 3D volume assembly from 2D SAM segments resolves the diagonal-object and stacked-object problems that a naive IoU threshold approach cannot handle.
  • SAM performs well on high-resolution data (35×728×968) but is less effective on lower-resolution volumes (120×120×120), where 3D Watershed or Region Growing are better suited.
  • Creating a fully modality-agnostic pipeline is challenging due to variation in data collection methods, resolution, and data shape. User-guided segmentation selection is the primary Phase II improvement.
  • SAM’s prompting capabilities (point, bounding box, natural language) offer a path toward an interactive assistant-style experience in Phase II.
  • Phase II will pivot to plugin/extension integration with existing microscopy platforms (Fiji/ImageJ, napari), focusing ScopeVis effort on the ML pipeline as the core differentiator.
